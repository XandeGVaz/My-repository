Observações em "Dados pesssoais":

1-Legenda com "for" e input com "id" permitem que ao clicar na legenda o campo de escrita tambéms seja ativado.
2-Operador "required" indica que o campo é obrigatório e o usuário não pode enviar o formulário sem prenchê-lo.
3-Veremos como indicar o arquivo para o qual as informações do formulário preenchido serão enviadas nas aulas de back-end, porém essas informações são indicadas geralmente dentro da tag "form".
4-Para deixar a marca de água que indica onde o usuário deve escrever algo usamos o operador " required="escreva ___ aqui" ".

Observações em "Linguagem preferida":

1-Para que a única escolha funcione, é necessário que todos os inputs tenham "type" radio e tenham o mesmo atributo "name".
2-O value é importante pois ele determina o valor que será enviado para o arquivo de resposta do formulário.

Observações em "Linguagens usadas":

1-Para que a múltipla escolha funcione, é necessário que todos os inputs tenham "type" checkbox e tenham atribustos "name" diferentes

Observações em "Melhor horário de rendimento":

1-Nesse caso não utilizamos a tag "input", mas sim a tag "select".
2-Podemos colocar uma seleção desabilitada que informa o usuário a necessidade de selecionar uma opção válida (observar a primeira opção).
3-O select tem que ter "name" para indicar o que será enviado para o arquivo no back-end.

Observações em "Sua rotina"

1-Assim como o input e o select, precisa de um atributo name.
2-Pode ser dimensionado usando os operadores "rows" e "cols".

Observações em "Clique"

1-Cria um botão genérico.
2-Se esse botão for colocado dentro do formulário, ele terá a mesma função de enviar o formulário, caso o contrário ele continuará genérico.
